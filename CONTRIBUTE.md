# 🌿 Contributing to friendly-arsenal

I'm really excited you’re interested in helping out. This project is a community effort, and I want to make it as easy and fun as possible for you to jump in.

## How to get started

### 1. Found a bug?
No worries, it happens! If you run into something that doesn't seem right, please open an [Issue]. To help us fix it faster, try to include:
* A quick description of what's happening.
* The steps you took that triggered the issue.
* What you expected to see vs. what actually happened.

### 2. Have a great idea?
Got an idea for a new feature or an improvement? I'd love to hear it. Please open an [Issue] to chat about it before you start coding—it helps me make sure we're all on the same page.

### 3. Submitting your code (Pull Requests)
When you're ready to submit a fix or feature:
1. **Fork** the repository to your own account.
2. Create a new **branch** for your work: `git checkout -b your-feature-name`.
3. **Commit** your changes with a message that makes it easy to see what you did.
4. **Push** your branch to your fork.
5. Open a **Pull Request (PR)** against our `main` branch. I'll take a look as soon as I can!

## Getting the project running locally
If you want to poke around or test your changes:
1. **Clone** the repo: `git clone [url]`
2. **Install** everything you need: `npm install`
3. **Set up** your environment: Copy `.env.example` to create your own `.env` file and add your keys.
4. **Fire it up**: Run `npm run dev` and you're good to go!

## A quick note on behavior
I want this to be a welcoming place for everyone. By contributing, you agree to be respectful and kind to your fellow developers. Let's keep it positive!
